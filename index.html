System Design

Program Architecture

DS
Data Segment
Buffers, '$'-terminated strings, GC counter, complement lookup table, and runtime flags — all defined in .DATA.
.MODEL SMALL → DS ≤ 64 KB
CS
Code Segment
MAIN PROC, menu loop, four feature routines, and helper PROCs — all within one flat code segment.
INT 21h for all I/O
SS
Stack Segment
256-byte stack declared with .STACK 100h. PUSH/POP preserves registers across PROC calls.
.STACK 100h = 256 bytes
AX
Registers Only
Only 16-bit registers: AX BX CX DX SI DI BP SP. No 32-bit extensions — pure 8086 constraint.
mov ax, @data / mov ds, ax

Feature 01

Complement
Generator

Each base is looked up in a pre-built Translation Table in the Data Segment. One memory read replaces 20 conditional branches.

Load base → use ASCII code as table index → write complement. Pure O(n) with zero branching per character.

complement_loop
1mov al, [si] ; load base → AL register
2xor bh, bh ; zero upper byte of BX
3mov bl, al ; BX = ASCII code of base
4lea bx, comp_table[bx] ; BX → table entry
5mov al, [bx] ; AL = complement char

Feature 02

GC-Content
Calculator

Counts G and C occurrences, then computes the percentage using 16-bit MUL and DIV.

Critical detail: MUL BX produces a 32-bit result in DX:AX — always zero DX before DIV.

gc_calc
1mov ax, gc_count ; AX = number of G/C bases
2mov bx, 100 ; scale factor
3mul bx ; DX:AX = gc_count × 100
4mov bx, seq_len ; denominator
5div bx ; AX = GC% integer result

Feature 03

3-Base Motif
Search

A sliding window of size 3 scans the main sequence. At each position three back-to-back CMP instructions check for a full match.

Uses SI and DI as pointer registers — the classic 8086 string-processing idiom.

motif_outer
1lea si, dna_buf ; SI → sequence start
2add si, found_idx ; offset by window position
3lea di, mot_buf ; DI → motif buffer
4mov al, [si] ; compare position 0
5cmp al, [di]
6jne MOTIF_NEXT ; mismatch → slide window

Defense Preparation

Viva / Defense Tips

01
◈
The ASCII Trick
Don't say "I change the letter." Say: "I move a byte from a segmented memory address into the AL register — that's a memory-to-register transfer with associated latency."
; A = 41h, T = 54h
mov al, [si] ; mem → reg
02
◈
The Lookup Table
Instead of chained IF/ELSE, a Translation Table in .DATA gives O(1) lookup per base. It's the Assembly-native pattern — fast, no branch mispredictions.
lea bx, comp_table[bx]
mov al, [bx] ; 1 mem read
03
◈
16-bit Segment Limit
A 16-bit segment is exactly 64 KB. Sequences beyond that boundary require segment switching. Keeping the buffer at 100 chars stays safely within one segment.
; 100 chars × 1 byte = 100 B
; Limit = 65,535 B → safe ✓
04
◈
MUL Overflow Risk
16-bit MUL produces a 32-bit result split across DX:AX. Failing to zero DX before DIV is the #1 bug in student GC calculators — mention this proactively.
mul bx ; DX:AX = result
div bx ; AX = quotient
05
◈
Stack Discipline
Every PUSH must be matched by a POP before RET. The print_word_decimal routine uses PUSH to reverse digit order — explain this LIFO property explicitly.
push dx ; save digit
loop …
pop dx ; LIFO order ✓
06
◈
Segment Init
On startup, CS is set by DOS but DS is not. The first two instructions — mov ax, @data / mov ds, ax — are mandatory. Without them, data access is undefined.
mov ax, @data
mov ds, ax ; MANDATORY
